Computation Registry

Restructured analytical engine: 96 patterns consolidated into unique computations with output facts. No judgmental thresholds. No family lock-in. Each computation produces facts that any diagnostic view can consume.

Foundational Principles

  1. Data equality — PM/CRM and field/audit data are co-equal primary sources
  2. Hybrid scoring — Full precision internally, 0-10 externally
  3. Three layers — Data layer (by domain), computation layer (flat), diagnostic views (by category)
  4. Separation of concerns — Computations produce facts. Scoring judges. No judgmental thresholds in computations.
  5. Multi-view outputs — One computation's facts can feed multiple diagnostic views simultaneously
