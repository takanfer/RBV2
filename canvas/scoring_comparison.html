Exclusive Coverage

Capabilities present in one system but absent from the other.

RBv1 Only

  • Audit & Field Validation (F10) — Scores contradiction control, confirmation strength, domain consistency from field audits
  • Longitudinal Progress (F12) — Tracks score changes, metric balance, persistent weakness reduction across assessments
  • Micro-Slice Diagnostics (F14) — Detects divergence by unit type, building/floor, and staff member
  • Door Opener Hypothesis (F15) — Anomaly density scoring for pre-engagement public-data assessments
  • Contradiction penalties — 0.85x multiplier + trust degradation when field and system data disagree
  • Score caps on missing data — Per-family max scores when critical dimensions are absent
  • 4-tier support + confidence model — Coverage %, contradiction ratio, sample size determine trust independently
  • Finding-only pattern classification — Patterns that inform findings but intentionally do not move scores
  • Cross-family pattern folding — VTP patterns placed in maintenance, retention, or marketing based on causal ownership

V6 Only

  • Overall property score — Single 0-10 number from weighted module rollup
  • Business tier + fee assignment — Optimization / Repositioning / Turnaround / Custom with fee levels
  • Technology & Systems (M14) — Coverage, depth, integration, automation scoring
  • Organization & Staffing (M15) — Training, coaching, staffing capacity
  • Management Quality (M16) — Maintenance + condition from assessment sheets
  • Resident Screening (M17) — Screening process quality
  • Pet Policy (M19) — Pet revenue and policy effectiveness
  • Industry benchmark citations — NAA 2024, NMHC/Grace Hill 2024
  • Assessment sheet integration — Reads scores from 6 Google Sheets assessment tabs
  • 200+ rubric VLOOKUP keys — Form-driven scoring for amenities, marketing, leasing, condition
  • VDOM anchor KPI — Dedicated vacancy-days-on-market anchor metric

Tradeoff Summary

Key considerations for choosing or merging the two systems.

◆ RBv1 Strengths

  • + Analytically deeper: 96 patterns vs ~66, more dimensions per family
  • + Handles uncertainty honestly: support, confidence, and score are separate
  • + Contradiction-aware: field-vs-system disagreement impacts scores
  • + Score caps prevent over-claiming when data is incomplete
  • + Cross-family pattern folding prevents double-counting
  • + Door opener and longitudinal families are first-class
  • + Micro-slice analysis catches building/unit-type/staff-level problems
  • + Config-driven thresholds (JSON) are deployment-friendly

◆ RBv1 Weaknesses

  • − No overall property score or tier classification
  • − No industry benchmark citations
  • − Cannot score Tech, Org, Staffing, Screening, Pet Policy
  • − More complex to explain to clients
  • − No form/rubric-based scoring for intake-driven assessments

◆ V6 Strengths

  • + Single property score and business tier with fee linkage
  • + Industry-cited benchmarks (NAA, NMHC/Grace Hill)
  • + Form-driven rubric scoring for intake/observation data
  • + Simpler 0-10 scale; easier client communication
  • + Covers technology, staffing, screening, management
  • + Working code exists (ScoringEngine.gs)
  • + Piecewise linear normalization is intuitive

◆ V6 Weaknesses

  • − No contradiction handling
  • − Simpler confidence model
  • − No score caps for missing data
  • − No longitudinal or door-opener scoring
  • − No micro-slice analysis
  • − 7 rubric sections have acknowledged gaps
  • − Coupled to Google Sheets (not API-first)

Decision Framework

V6 was built for a Google Sheets workflow: intake forms, rubric VLOOKUP keys, and quick module scores for a working engagement tool. RBv1 was designed as the analytical engine for a purpose-built platform: config-driven, pattern-based, with explicit uncertainty handling.

Key questions to decide:

1. Do you need an overall property score and tier? If yes, the new system needs a cross-family roll-up added to RBv1's 15 families (straightforward) or V6's weighted module approach adopted.

2. Do you need rubric-based intake scoring? V6's amenity, marketing, leasing ops, and technology rubrics capture information not from PM/CRM data. If those assessments matter, the rubric system needs to coexist with the analytical engine.

3. Which scale for client communication? 0–100 with letter grades is common in education; 0–10 is simpler. The tier names (Optimization / Repositioning / Turnaround) carry business meaning. Either scale works; the choice is presentation.

4. Is contradiction handling essential? If field audits are a core differentiator of your methodology, then RBv1's contradiction penalties and trust model are not optional.

A hybrid approach would use RBv1's 15-family analytical engine (96 patterns, contradiction handling, confidence model) as the core, add V6's rubric-based scoring for intake/observation domains (tech, staffing, amenities), define an overall property roll-up, and adopt V6's benchmark citations where available.
